Detailed Description

This Structure Will be parsed as a Unicast Message. The
communication will have this format. Each drone receives
Unicast Message in this format. The parser will convert
the received message into struct members.
.

  • source_id (int) - The ID which I have while Sending the message
  • dst_id (int) - The Destination ID, Message will be sent to this ID
  • nbs (string) - The Neighbour Table Of this Node,
    Format of String is | seperated as follows:
    (.. | ID(int), hop(int), pos(ns3::Vector) | ..)
    hop - 1,2,3 ...
    pos - x (float) : y (float) : z (float)
  • control - This is a bit field which consists of all possible
    control commands a particular drone can give.
    1 -> Drone Should Unarm and Land
    2 -> Drone Should Hold Pos at RC from Parent
    4 -> Drone Should Change Parent to p_id
    8 -> Drone Should Go Left towards the p_id
    16 -> Drone Should Go Right towards the p_id
    32 -> Drone Should Go Left towards the boundary
    64 -> Drone Should Go Right towards the boundary
  • state - This is a bit field consisting of the state this
    drone is in.
    1 -> Fire Detected
    2 -> On Straight Line (Phase 1)
    4 -> Circling Around Anchor (Phase 2)
    8 -> Land and Unarmed
    16 -> Global Fire Detected
    32 -> Drone Required towards the boundary
    64 -> Reserved (Global)
  • p_id - Parent To Follow
